# Feature: F01-05 前端路由结构与基础布局

> 版本：v1.0 | 日期：2026-03-16
> 状态：Phase 1 完成

---

## 背景

F01-04 已完成前端 Monorepo 搭建，创建了两 Next.js 15 应用的基础结构。现在需要为 web 和 admin 应用添加路由组和基础布局，为后续业务页面开发提供可用的导航框架。

本项目采用"按需配置路由"原则，不预先创建空路由页面。

---

## 目标

- 在 web 应用中创建 `(public)` 和 `(auth)` 路由组
- 为 web 应用添加顶部导航栏（占位）
- 为 admin 应用添加侧边导航栏（占位）
- 添加 404 页面处理
- 确保所有配置的路由可访问且无报错

---

## 范围

### 包含（In Scope）

- **Web 应用**
  - `(public)` 路由组：首页
  - `(auth)` 路由组：登录页
  - 根布局：全局容器
  - 顶部导航栏（占位）
  - 404 页面

- **Admin 应用**
  - 根布局：侧边导航容器
  - 首页
  - 侧边导航栏（占位）
  - 404 页面

- **共享**
  - 基础样式复用
  - 应用名称确定（web: "AI 研云"，admin: "运营后台"）

### 不包含（Out of Scope）

- 其他路由（chat、agents、creative 等）留到对应业务 Feature
- shadcn/ui 组件（F01-06）
- Tailwind CSS 配置（F01-06）
- 具体导航菜单项（无路由则无菜单）
- 路由保护和鉴权逻辑

---

## 验收标准（Acceptance Criteria）

### AC1: Web 应用路由组结构
- **验收方式**：`ls web/src/app/\(public\)/` 和 `ls web/src/app/\(auth\)/`
- **期望**：存在 `(public)` 和 `(auth)` 目录，各含 layout.tsx 和 page.tsx

### AC2: Web 应用首页可访问
- **验收方式**：`pnpm --filter web dev` + 访问 `http://localhost:3000`
- **期望**：显示 "AI 研云" 标题，无报错

### AC3: Web 应用登录页可访问
- **验收方式**：访问 `http://localhost:3000/login`
- **期望**：显示登录页标题，无报错

### AC4: Web 应用包含顶部导航占位
- **验收方式**：查看首页源码/渲染结果
- **期望**：可见顶部导航区域

### AC5: Admin 应用包含侧边导航占位
- **验收方式**：`pnpm --filter admin dev` + 访问 `http://localhost:3001`
- **期望**：显示 "运营后台" 和侧边导航区域

### AC6: 404 页面存在
- **验收方式**：访问不存在的路由（如 `http://localhost:3000/nonexistent`）
- **期望**：显示 404 页面，包含 "页面不存在" 文字和返回首页链接

### AC7: 所有路由无报错
- **验收方式**：`pnpm -r typecheck` + `pnpm --filter web build` + `pnpm --filter admin build`
- **期望**：类型检查通过，构建成功

---

## 约束

### 技术约束
- Next.js 15 App Router 路由组语法
- 不引入新的 CSS 框架
- 复用 F01-04 的 `globals.css`

### 架构约束
- 路由组 `(public)` 和 `(auth)` 不出现在 URL 中
- 根布局只包含全局容器，导航由路由组布局添加
- 按需创建路由，不预建空页面

### 设计约束
- 导航栏为占位组件，只显示文字标识
- 页面内容为简单标题，暂无交互逻辑
- 404 页面使用简单的 HTML 结构
