## 一、测试策略

### 1.1 测试类型

本 Feature 为前端路由结构和布局组件，测试方式以验证为主：

| 测试类型 | 验证方式 | 工具 |
|----------|----------|------|
| 配置验证 | 文件存在性、目录结构 | ls, cat |
| 类型检查 | TypeScript 编译 | tsc --noEmit |
| 构建验证 | Next.js 构建 | next build |
| 路由验证 | 访问路由确认渲染 | next dev（手动） |

### 1.2 不适用项

- 本 Feature 不适用单元测试（纯布局组件，无业务逻辑）
- 不适用集成测试（无后端交互）
- 不适用 PIT 变异测试（配置文件）
