# Epic 2: 用户与登录

> 版本：v1.0 | 日期：2026-03-18
> 状态：待实施
> 预估总复杂度：M（中等）

---

## 一、背景

实现用户注册、登录、个人信息管理功能，为 AI 对话等核心功能提供用户身份基础。

本 Epic 完成后，系统应具备：
- 用户可通过邮箱或手机号注册/登录
- 个人用户自动创建单人租户（Tenant）
- 用户可管理个人信息（昵称、头像等）
- web 端和 admin 端的认证页面与状态管理
- 为后续 Epic（AI 对话、计费、多租户）提供 Account Context 基础

---

## 二、范围

| 类别 | 包含 | 不包含 |
|------|------|--------|
| **后端** | Account Context（用户注册/登录/个人信息）、Tenant Context（单人租户创建）、认证基础设施 | API Key 管理、企业/租户管理（Epic 6）、OAuth 第三方登录 |
| **前端** | web 端登录/注册/个人中心页面、认证状态管理、admin 端登录页 | 个人中心的 API Key 管理、企业管理视图（/manage/*） |
| **安全** | 密码加密、JWT Token、验证码防刷限流 | 多因素认证、单点登录（SSO） |

## 三、Feature 清单

### F02-01: User 聚合根与领域模型（复杂度：S，无依赖）

建立 User 聚合根及相关领域模型，包括 Email/PhoneNumber 值对象、密码加密策略。

**验收标准：**
- User 聚合根包含：id、email/phone、password（加密）、nickname、avatar、createdAt、updatedAt
- Email 值对象：格式校验、唯一性
- PhoneNumber 值对象：格式校验（11位手机号）、唯一性
- 密码使用 BCrypt 加密存储
- UserRepository 接口定义
- 单元测试覆盖值对象校验逻辑

---

### F02-02: 用户注册 - 邮箱验证码服务（复杂度：M，依赖 F02-01）

实现邮箱验证码发送与校验，支持注册和密码重置场景。

**验收标准：**
- 发送验证码接口：POST /api/account/verification-code/email
- 验证码 6 位数字，有效期 5 分钟
- 同一邮箱 60 秒内只能发送一次
- 同一 IP 每小时最多发送 10 次
- 验证码校验接口：POST /api/account/verify-code
- Redis 存储验证码，支持过期自动清理
- 单元测试 + 集成测试覆盖限流场景

---

### F02-03: 用户注册 - 邮箱流程（复杂度：M，依赖 F02-01, F02-02）

实现邮箱注册完整流程，包括验证码校验、密码校验、创建 User 和单人租户。

**验收标准：**
- 注册接口：POST /api/account/register/email
- 请求字段：email、password、nickname、verificationCode
- 邮箱唯一性检查
- 密码强度校验（8-20 位，包含字母和数字）
- 注册成功自动创建 Personal 类型租户
- 返回 JWT Token
- 单元测试覆盖：成功、邮箱重复、验证码错误、密码弱

---

### F02-04: 用户注册 - 手机号流程（复杂度：M，依赖 F02-01, F02-02）

实现手机号注册完整流程，包括短信验证码、创建 User 和单人租户。

**验收标准：**
- 注册接口：POST /api/account/register/phone
- 请求字段：phone、password、nickname、verificationCode
- 手机号唯一性检查
- 短信验证码发送（对接短信服务商，初期可模拟）
- 注册成功自动创建 Personal 类型租户
- 返回 JWT Token
- 单元测试覆盖：成功、手机号重复、验证码错误

---

### F02-05: 用户登录（复杂度：M，依赖 F02-01, F02-10）

实现邮箱/手机号密码登录，生成 JWT Token。

**验收标准：**
- 登录接口：POST /api/account/login
- 支持邮箱或手机号作为账号
- 密码错误 5 次锁定账号 30 分钟
- 登录成功返回 JWT Token（有效期 7 天）
- 记录登录日志（IP、时间、设备）
- 单元测试覆盖：成功、账号不存在、密码错误、账号锁定

---

### F02-06: 密码重置（复杂度：M，依赖 F02-01, F02-02）

实现忘记密码功能，通过验证码重置密码。

**验收标准：**
- 发送重置验证码：复用 F02-02 验证码服务
- 重置接口：POST /api/account/reset-password
- 请求字段：account（email/phone）、verificationCode、newPassword
- 新密码强度校验
- 重置成功后旧 Token 失效
- 单元测试覆盖：成功、验证码错误、账号不存在

---

### F02-07: 个人信息管理（复杂度：S，依赖 F02-01, F02-10）

实现个人信息查询与更新。

**验收标准：**
- 查询接口：GET /api/account/profile
- 更新接口：PUT /api/account/profile
- 可更新字段：nickname、avatar
- 鉴权：需登录
- 单元测试覆盖：查询、更新、未授权访问

---

### F02-08: 单人租户自动创建（复杂度：S，依赖 F02-01）

个人用户注册时自动创建 Personal 类型租户。

**验收标准：**
- TenantService.createPersonalTenant(userId) 方法
- 租户名称默认为用户昵称
- 租户类型为 PERSONAL
- 用户自动成为该租户的 OWNER
- 单元测试覆盖：创建、用户-租户关联

---

### F02-09: 认证基础设施（复杂度：M，依赖 F02-01）

配置 Spring Security 和 JWT 过滤器，提供 `@CurrentUser` 注解。

**验收标准：**
- JWT 过滤器：从 Bearer Token 解析用户 ID
- SecurityConfig 配置：公开路径（注册、登录）需认证路径
- `@CurrentUser` 注解：Controller 方法直接注入当前用户 ID
- 无效 Token 返回 401
- 单元测试覆盖：有效 Token、无效 Token、过期 Token

---

### F02-10: web 端登录页/注册页（复杂度：M，依赖 F02-01~F02-06, F02-12）

实现登录和注册页面，支持邮箱/手机号切换。

**验收标准：**
- 路由：/login、/register
- Tab 切换：邮箱登录/手机号登录
- 注册表单：邮箱/手机号、密码、昵称、验证码
- 表单验证：实时校验 + 错误提示
- 登录成功跳转首页
- 注册成功自动登录并跳转首页
- 使用 shadcn/ui 组件（Button、Input、Label、Card）

---

### F02-11: web 端认证状态管理（复杂度：M，无依赖）

使用 Zustand 管理认证状态，实现 Token 存储与刷新、路由守卫。

**验收标准：**
- AuthStore：token、user、isAuthenticated
- Token 存储：localStorage
- 路由守卫：未登录跳转 /login
- Token 过期自动跳转 /login
- 登出方法：清除 Token + 跳转 /login
- 单元测试覆盖：登录、登出、Token 过期

---

### F02-12: web 端个人中心页面（复杂度：M，依赖 F02-07, F02-11）

实现个人中心页面，展示和编辑个人信息。

**验收标准：**
- 路由：/account
- 展示：邮箱/手机号、昵称、头像、注册时间
- 编辑：昵称、头像上传
- 密码修改：旧密码 + 新密码 + 确认密码
- 导航：从头像菜单进入
- 使用 shadcn/ui 组件

---

### F02-13: admin 端登录页（复杂度：S，依赖 F02-09, F02-11）

实现 admin 应用的简化登录页。

**验收标准：**
- 路由：/login
- 仅支持邮箱登录
- 复用 web 端 AuthStore
- 登录成功跳转 /tenants 首页
- 使用 shadcn/ui 组件
