## 一、平台概述

### 1.1 产品定位

海创元智研云平台是一个面向学校和企业的 **AI 聚合 SaaS 平台**，核心价值：

- **模型统一接入**：一次接入，使用多家大模型（GPT、Claude、文心一言等），无需分别注册
- **智能体市场**：预制和自定义 AI 智能体，处理复杂任务（PPT 生成、文档分析、编程助手等）
- **费用统一管理**：虚拟币计费，企业配额控制，用量统计透明
- **多租户支持**：企业/机构入驻，成员管理，权限配置

### 1.2 产品形态

| 形态 | 说明 | 访问地址 |
|------|------|----------|
| **用户端** | 面向普通用户和企业管理员 | `http://localhost:3000`（本地） |
| **运营后台** | 面向平台运营人员 | `http://localhost:3001`（本地） |
| **API 服务** | RESTful API，供前端调用 | `http://localhost:8080`（本地） |

### 2.2 限界上下文

系统按业务领域划分为 8 个限界上下文（Bounded Context）：

| 上下文 | 职责 | 主要功能 |
|--------|------|----------|
| **Account** | 用户账户 | 注册、登录、个人信息、API Key |
| **Tenant** | 租户管理 | 企业入驻、成员管理、角色权限、配额控制 |
| **Gateway** | AI 网关 | 模型路由、请求调度、限流降级、调用日志 |
| **Conversation** | 对话管理 | 会话列表、消息存储、上下文窗口管理 |
| **Billing** | 计费系统 | 虚拟币余额、充值/退款、消耗计费、汇率管理 |
| **Agent** | 智能体 | 智能体定义、工具注册、执行引擎、多步编排 |
| **Creative** | 创作工具 | 生图、生音视频、AI 编程 |
| **Admin** | 平台管理 | 模型/Provider 管理、定价汇率、运营数据、审计 |

## 三、系统操作指南

### 3.1 启动系统

#### 方式一：一键启动（推荐）

```bash
./scripts/dev.sh
```

此命令会：
- 后台启动 Spring Boot 后端服务（8080 端口）
- 前台启动 Next.js Web 前端（3000 端口）
- Ctrl+C 时自动清理后端进程

#### 方式二：分别启动

```bash
# 后端
cd server && ./gradlew bootRun

# Web 前端（新终端）
pnpm dev:web

# Admin 前端（新终端，可选）
pnpm dev:admin
```

### 3.2 访问系统

| 系统 | 本地地址 | 说明 |
|------|----------|------|
| 用户端 | http://localhost:3000 | 面向普通用户和企业管理员 |
| 运营后台 | http://localhost:3001 | 面向平台运营人员 |
| API 文档 | http://localhost:8080/swagger-ui.html | OpenAPI/Swagger 文档 |
| 健康检查 | http://localhost:8080/actuator/health | 系统健康状态 |

### 3.3 停止系统

```bash
# 如果使用 dev.sh 启动，直接 Ctrl+C

# 如果分别启动，手动停止各进程
# 后端：Ctrl+C 或 ./gradlew --stop
# 前端：Ctrl+C
```

### 3.4 健康检查

```bash
# 后端健康检查
curl http://localhost:8080/actuator/health
# 预期返回：{"status":"UP"}

# 应用级健康检查
curl http://localhost:8080/api/health
# 预期返回包含 status 和 timestamp
```

## 四、API 接口说明

### 4.1 健康检查接口

#### 应用级健康检查

```
GET /api/health
```

**响应示例**：
```json
{
  "code": 200,
  "message": "Success",
  "data": {
    "status": "ok",
    "timestamp": "2026-03-16T12:34:56.789Z"
  },
  "requestId": null
}
```

#### Actuator 健康检查

```
GET /actuator/health
```

**响应示例**：
```json
{
  "status": "UP"}
```

### 4.2 OpenAPI 文档

| 端点 | 说明 |
|------|------|
| `/swagger-ui.html` | Swagger UI 交互式文档 |
| `/v3/api-docs` | OpenAPI 3.1 规范 JSON |

**认证方式**：Bearer Token（JWT）
```
Authorization: Bearer {token}
```

### 4.3 API 端点规范

所有业务 API 遵循以下规范：

- **基础路径**：`/api`
- **版本**：`/v1`
- **响应格式**：统一的 `ApiResponse<T>` 结构
- **错误处理**：标准错误码和错误消息

**示例**：
```
GET /api/v1/conversations
POST /api/v1/tenants/{id}/members
PUT /api/v1/billing/coins/deduct
```

## 六、部署说明

### 6.1 部署方式

系统支持多种部署方式：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **本地开发** | 直接运行后端和前端 | 开发、测试 |
| **Docker Compose** | 容器化部署 | 开发环境、测试环境 |
| **生产部署** | Docker/Kubernetes | 生产环境 |

### 6.2 Docker 部署

#### 开发环境

```bash
./scripts/deploy-dev.sh
```

此命令会：
- 验证代码
- 构建 Docker 镜像
- 使用 `docker-compose.dev.yml` 启动服务
- 包含：PostgreSQL、Redis、Backend、Web、Admin

#### 生产环境

```bash
./scripts/deploy-prod.sh
```

**注意事项**：
- 需要配置 `.env.prod` 环境变量文件
- 部署前需要确认
- 不包含数据库（使用外部数据库）

### 6.3 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `SERVER_PORT` | 后端端口 | 8080 |
| `DB_URL` | 数据库连接 | - |
| `REDIS_URL` | Redis 连接 | - |
| `NEXT_PUBLIC_API_URL` | 前端调用后端地址 | `http://localhost:8080` |

## 七、故障排查

### 7.1 常见问题

#### 后端启动失败

**症状**：`./gradlew bootRun` 报错

**检查项**：
1. Java 版本是否为 21+
2. cartisan-boot 是否在 `../cartisan-boot/` 目录
3. 端口 8080 是否被占用

#### 前端启动失败

**症状**：`pnpm dev:web` 报错

**检查项**：
1. Node.js 版本是否为 20+
2. pnpm 版本是否为 9+
3. `pnpm install` 是否已执行
4. 端口 3000 是否被占用

#### 前端无法调用后端

**症状**：前端请求后端 API 报错

**检查项**：
1. 后端是否正在运行
2. Next.js rewrites 配置是否正确
3. 跨域配置是否正确
